# Version 1.4.0
- Added pluggable `SessionStore` support to `Middleware.session()` and exposed `Middleware.createMemorySessionStore()` as the default in-memory backend helper
- Added `Middleware.createRedisSessionStore()` for Bun-native Redis-backed production session storage
- Added `Middleware.csrf()` with session-backed tokens, automatic `req.getCsrfToken()` access, and Ox runtime header propagation for mutating `ox-*` requests
- Added hashed Ox runtime asset versioning so `router.serveOx()` returns an immutable `/__oxarion/ox.<hash>.js` path while keeping `/__oxarion/ox.js` as a stable alias
- Added browser E2E coverage for the session-backed Ox counter flow and unit coverage for custom session stores, CSRF protection, and hashed runtime serving
- Added a dedicated `benchmarks/run-ssr-runtime-benchmark.ts` harness and `benchmarks/OxRuntimeResults.md` report for SSR pages, fragments, and the hashed Ox runtime asset
- Added a gated Docker-backed Redis integration test and extended the browser smoke flow to validate hashed runtime cache headers and CSRF token exposure
- Extended the SSR/runtime benchmark harness with a stateful session + CSRF protected fragment mutation endpoint
- Added `docs/production_deployment.md` with production guidance for Redis sessions, CSRF ordering, cookie settings, and hashed runtime asset usage
- Optimized static `OxarionResponse.text()`, `.json()`, and `.html()` to skip `Headers` allocation when no custom init headers are provided
- Optimized `OxarionResponse` instance creation by lazily allocating response headers only when they are actually used
- Optimized `OxarionRequest.getQueries()` to cache parsed query parameters per request
- Optimized adapter request handling to reuse the parsed pathname instead of reparsing it during route matching
- Added support for returning instance response helpers like `return res.json(...)`, `return res.text(...)`, and `return res.html(...)` from handlers
- Updated docs to make `return res.json(...)` and other instance return helpers official API usage
- Added `Oxarion.create()` app instances with `app.start()`, `app.stop()`, `app.request()`, and `app.getRouter()`
- Added `router.mount()` as a clearer alias for `injectWrapper()`
- Added scoped service registration with `router.provide()` and request access via `req.getService()` / `req.hasService()`
- Added lifecycle hooks with `router.hook()` for `onRequest`, `preHandler`, `onSend`, `onResponse`, and `onError`
- Added async plugin registration with `router.register()`
- Added response streaming helpers `res.stream()` and `res.sse()`
- Added cached SSR template rendering with `res.render()`, `res.renderFragment()`, `app.render()`, and `app.renderFragment()`
- Added optional Ox dynamic HTML runtime via `router.serveOx()` and `/__oxarion/ox.js` with `ox-get`, `ox-post`, `ox-put`, `ox-delete`, `ox-target`, `ox-swap`, and `ox-trigger`
- Expanded the Ox dynamic HTML runtime with `ox-confirm`, `ox-include`, response swap overrides, and `window.Ox.apply()` / `window.Ox.swap()` helpers for SSE or manual DOM updates
- Refactored dynamic HTML placement around `ox-anchor`, `ox-place`, and `ox-mode`, while keeping `ox-target` and `ox-swap` as low-level compatibility fallbacks
- Refactored HTTP internals to share one request executor between Bun server fetch handling and `app.request()`
- Grouped helper exports under `OpenAPI.*` and `WebSocket.*` namespaces
- Added JSDoc coverage for the public type surface in `src/types.ts` to improve editor hints and API discoverability
- Updated `docs/route_injector.md` to document `router.mount()` as the preferred wrapper mounting API
- Added a one-time deprecation warning for `router.injectWrapper()` and marked it deprecated in the type surface ahead of `1.5.x`
- Added dedicated docs for SSR templates and dynamic HTML runtime behavior
